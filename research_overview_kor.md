# Research Vision

## 1. 연구 목표

내가 장기적으로 관심 있는 문제는, 생성이 끝난 뒤에 외부 평가 모델을 붙여서 점수를 매기는 수준을 넘어서, 생성 과정 안에 외부 평가 정보를 자연스럽게 넣는 것이다.

특히 text-to-image 생성에서 사용하는 diffusion 모델이나 flow matching 모델은 하나의 결과를 한 번에 내는 방식이 아니라, 시간에 따라 샘플이 조금씩 변해 가는 과정으로 볼 수 있다. 나는 이 과정을 단순한 샘플링 절차가 아니라, 중간에 점검하고 수정할 수 있는 동적인 생성 과정으로 보고 싶다.

지금 관심이 있는 외부 평가는 주로 vision-language model(VLM)이다. 이런 모델은 생성 결과가 프롬프트와 잘 맞는지, 속성이 빠지지는 않았는지, 관계가 어색하지 않은지 같은 문제를 꽤 잘 잡아낸다. 다만 이런 평가는 보통 미분이 되지 않고, 생성 중간 단계에서는 신호가 불안정할 수 있다는 어려움이 있다.

내가 궁극적으로 만들고 싶은 것은, 생성 도중의 상태를 보고 현재 결과의 문제를 어느 정도 판단하고, 그 판단을 바탕으로 이후 생성 궤적을 수정해 나갈 수 있는 생성 모델이다. 쉽게 말하면, 그냥 이미지를 뽑아내는 모델이 아니라, 중간에 스스로 점검하고 수정할 수 있는 생성 방식을 연구하고 싶다.

## 2. 문제의식

최근의 text-to-image 모델은 이미지 품질 자체는 많이 좋아졌지만, 여전히 프롬프트를 완전히 정확하게 따르지 못하는 경우가 많다. 예를 들어 특정 속성이 빠지거나, 객체 사이의 관계가 틀리거나, 개수가 맞지 않거나, 전체 구성이 프롬프트 의도와 어긋나는 경우가 있다.

이런 문제는 종종 생성 모델 내부 신호만으로는 잘 드러나지 않는다. 반면 VLM 같은 외부 모델은 이런 오류를 더 잘 감지할 때가 많다. 그래서 최근에는 생성이 끝난 뒤 VLM이나 reward model로 점수를 매기고, 여러 샘플 중 좋은 결과를 고르는 방식이 많이 쓰인다.

하지만 이런 방식은 결국 생성이 끝난 다음에만 개입할 수 있다는 한계가 있다. 이미 잘못된 방향으로 생성이 진행된 뒤라면, 마지막에 점수를 잘 매겨도 근본적인 수정은 어렵다. 그래서 나는 외부 평가를 마지막 선택 단계에만 쓰는 것이 아니라, 생성 중간에 더 적극적으로 활용하는 방향이 필요하다고 생각한다.

## 3. 내가 보는 연구 관점

나는 diffusion과 flow matching을 조금 넓게 보면 둘 다 trajectory-based generative model로 볼 수 있다고 생각한다.

diffusion에서는 노이즈가 많은 상태에서 시작해서 점차 이미지를 복원해 가는 과정을 따라간다. flow matching에서는 학습된 velocity field를 따라 연속적으로 샘플이 이동한다고 볼 수 있다. 표현은 다르지만, 둘 다 생성이 시간에 따라 전개되는 과정이라는 점에서는 비슷하다.

이 관점에서 핵심 질문은 다음과 같다.

외부 평가 모델의 판단을 어떻게 생성 궤적 수준의 수정 신호로 바꿀 수 있을까?

이 질문 안에는 몇 가지 하위 문제가 같이 들어 있다. 생성 중간 상태를 어떤 기준으로 평가할 수 있는지, 어떤 시점의 평가가 믿을 만한지, 최종적으로 드러난 오류를 생성 과정의 어느 부분과 연결할 수 있는지, 그리고 이런 평가를 실제로 guidance나 control signal처럼 쓸 수 있는지가 모두 중요한 문제다.

## 4. 현재까지의 출발점

현재까지 내가 해 온 연구는 training-free setting에서 text-to-image diffusion의 추론 과정을 개선하는 쪽에 가깝다. 특히 초반에는 semantic한 구조를 좀 더 안정적으로 잡아 주고, 후반에는 외부 reward를 이용해 선택이나 resampling을 하는 방향을 다뤘다.

이 과정에서 느낀 가장 중요한 점은, 생성의 모든 단계가 같은 역할을 하지 않는다는 것이다. 초반에는 전체적인 의미 구조, 레이아웃, 객체 관계 같은 것이 아직 많이 바뀔 수 있지만, 후반으로 갈수록 perceptual quality나 전체적인 시각적 완성도를 평가하기가 더 쉬워진다.

그래서 외부 평가를 아무 시점에서나 같은 방식으로 쓰기보다는, 생성 단계에 따라 다르게 다뤄야 한다는 생각을 하게 됐다. 지금까지의 연구는 이런 문제를 다루는 첫 단계라고 볼 수 있다.

## 5. 앞으로의 확장 방향

앞으로는 이 문제를 diffusion에만 한정하지 않고, flow matching이나 더 일반적인 ODE 기반 생성 과정까지 넓혀서 보고 싶다.

이렇게 확장하고 싶은 이유는 세 가지 정도다.

첫째, ODE 관점에서는 생성 궤적이 연속적으로 어떻게 움직이는지를 조금 더 명확하게 볼 수 있다. 그래서 중간 개입이나 trajectory correction 문제를 더 깔끔하게 생각할 수 있다.

둘째, flow matching 계열은 velocity field 자체를 다룬다는 점에서, 외부 평가 정보를 단순한 점수로 쓰는 것을 넘어서 생성 방향 자체를 조정하는 문제와 연결하기 좋다.

셋째, 이런 관점은 diffusion sampling, control, feedback, test-time scaling 같은 주제를 하나의 흐름으로 묶어서 볼 수 있게 해 준다.

결국 내가 궁금한 것은, 외부 평가 모델이 어떤 샘플이 좋은지만 말해 주는 역할에 머무는 것이 아니라, 생성이 진행되는 방향 자체를 바꾸는 데까지 들어갈 수 있느냐는 점이다.

## 6. 핵심 연구 질문

앞으로의 연구는 대략 아래 질문들을 중심으로 정리할 수 있다.

첫째, VLM처럼 미분되지 않는 외부 평가 모델을 생성 과정 안에 어떻게 자연스럽게 넣을 수 있을까.

둘째, intermediate state를 어떤 방식으로 평가해야 실제로 의미 있는 신호를 얻을 수 있을까.

셋째, 생성 과정의 어느 시점부터 외부 평가 신호를 비교적 신뢰할 수 있을까.

넷째, 최종 결과에서 발견된 오류를 생성 과정의 어느 부분과 연결해서 볼 수 있을까.

다섯째, 외부 평가를 실제 corrective signal로 바꾼다면, 그것은 guidance 조정, latent 수정, branching, resampling, 또는 vector field control 중 어떤 형태가 될 수 있을까.

여섯째, VLM의 판단을 좀 더 부드럽고 계산 가능한 형태로 근사하는 surrogate model을 만들 수 있을까.

일곱째, 최종적으로 생성 모델이 단순한 sampler를 넘어서, 생성 중간에 오류를 진단하고 수정하는 self-corrective system으로 발전할 수 있을까.

## 7. 단계별 연구 방향

이 연구는 한 번에 끝나는 문제가 아니라, 몇 단계로 나눠서 발전시키는 쪽이 맞다고 생각한다.

### Stage 1

가장 현실적인 출발점은 외부 평가를 생성 후반이나 최종 단계에서 selection 용도로 사용하는 것이다. 여러 샘플이나 trajectory 중에서 더 나은 결과를 고르는 방식이다. 지금까지의 내 연구도 여기에 가장 가깝다.

### Stage 2

그다음 단계는 생성 중간에도 evaluator를 쓰되, 아무 때나 쓰는 것이 아니라 신뢰도가 올라가는 시점을 보고 제한적으로 개입하는 것이다. 예를 들어 후기 단계에서만 reward를 활성화하거나, evaluator reliability를 보고 resampling 여부를 정하는 식이다.

### Stage 3

그보다 더 나아가면 evaluator가 단순히 점수를 주는 역할을 넘어서, 생성 궤적을 직접 수정하는 방향으로 갈 수 있다. diffusion에서는 guidance를 조정하거나 latent 방향을 바꾸는 방식이 될 수 있고, flow matching에서는 velocity field나 control term에 개입하는 방식이 될 수 있다.

### Stage 4

마지막 단계는 생성 도중 intermediate output을 보고, 어떤 문제가 있는지 진단하고, 그에 맞춰 생성 과정을 다시 수정하는 self-corrective generation이다. 이 단계에서는 생성이 더 이상 고정된 절차가 아니라, 평가와 수정이 반복되는 과정이 된다.

## 8. Near-term, Mid-term, Long-term

### Near-term

단기적으로는 training-free setting에서 외부 evaluator를 diffusion sampling에 안정적으로 넣는 방법을 더 다듬고 싶다. 특히 late-stage reward-guided selection, evaluator reliability 분석, premature collapse를 줄이는 selection rule 같은 문제를 더 정리하고 싶다.

또한 어떤 timestep에서 어떤 종류의 evaluator signal이 의미를 가지는지 좀 더 체계적으로 분석해 보고 싶다.

### Mid-term

중기적으로는 최종 점수 기반 선택에서 한 단계 더 나아가, intermediate trajectory correction으로 가고 싶다. 예를 들면 속성 누락, 관계 오류, 개수 오류, compositional failure 같은 문제를 구분해서 보고, 그런 진단 결과를 바탕으로 생성 도중 수정 신호를 만드는 방향이다.

이 단계에서는 diffusion뿐 아니라 flow matching, continuous-time ODE 기반 생성으로도 문제를 확장해 보고 싶다. 외부 평가 신호를 단순한 score가 아니라 trajectory control signal처럼 해석하는 쪽에 관심이 있다.

### Long-term

장기적으로는 self-reflective, self-corrective generative system을 만드는 것이 목표다. 이를 위해서는 VLM 판단을 근사하는 differentiable surrogate model, step-wise diagnosis-and-revision loop, evaluator-informed generative dynamics 같은 요소가 필요하다고 본다.

궁극적으로는 생성 중간 상태를 보고 무엇이 잘못되었는지 판단하고, 최종 결과가 나오기 전에 trajectory를 수정해 가면서 더 나은 결과로 수렴하는 생성 시스템을 만들고 싶다.

## 9. 장기 비전

내가 생각하는 생성 모델의 다음 단계는, 단순히 더 큰 모델이나 더 많은 데이터로 성능을 올리는 것만은 아니다. 추론 시점에서 현재 생성 상태를 평가하고, 문제를 진단하고, 그 결과를 바탕으로 다음 생성을 수정하는 능력도 중요해질 것이라고 생각한다.

이런 관점에서 나는 generative model을 고정된 sampling procedure가 아니라, 중간에 관찰하고 조정할 수 있는 동적인 시스템으로 보고 싶다. diffusion, flow matching, ODE 기반 생성, external evaluator integration, test-time scaling은 결국 이 큰 흐름 안에서 연결될 수 있다고 본다.

정리하면, 내가 장기적으로 하고 싶은 연구는 외부 평가 신호를 생성 과정 안에 녹여 넣고, 그것을 바탕으로 생성 궤적을 점검하고 수정할 수 있는 모델을 만드는 것이다. 최종적으로는 단순히 이미지를 생성하는 모델이 아니라, 생성 과정에서 스스로 문제를 인식하고 더 나은 방향으로 수정해 갈 수 있는 모델을 목표로 하고 있다.
